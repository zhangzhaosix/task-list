# PROJECT AGENTS（强制执行）

## 1. 项目初始化（最高优先级）
进入项目后，任何开发行为前必须完成：
1. 阅读 `README.md`
2. 扫描项目结构
3. 理解项目用途、技术栈、模块职责
4. 检查 `AGENTS.md`
   - 不存在则创建
   - 存在则校验并更新
5. 将项目架构、核心模块、技术栈同步至 `AGENTS.md`

未完成以上步骤前禁止：
- 修改代码
- 创建文件
- 重构项目
- 开始开发

必须先完成项目认知，再进入设计与开发流程。

## 5. 编码原则
### 先思考后编码
必须：
- 明确假设
- 说明歧义
- 提供可选方案

存在关键不确定性时必须先提问。

禁止猜测需求后直接实现。

### 简洁优先
原则：
- 最少代码
- 最少复杂度
- 最少抽象

禁止：
- 过度设计
- 过度封装
- 提前开发未来需求

### 精准修改
仅允许修改与当前需求直接相关内容。

禁止：
- 顺手重构
- 顺手优化
- 修改无关模块

历史问题：
- 可以指出
- 不主动修改

## 项目认知（当前仓库）
- 项目类型：单页静态前端应用
- 入口文件：`index.html`
- 核心用途：每日任务清单管理
- 技术栈：`HTML`、`CSS`、原生 `JavaScript`、`Firebase Auth`、`Firestore`
- 运行方式：直接打开 `index.html` 即可使用，无需额外构建步骤
- 结构特征：当前仓库以单文件页面为核心，样式与脚本内联在 `index.html` 中
- 依赖特征：通过 CDN 引入字体与 Firebase 兼容版 SDK
- 核心模块：任务添加、任务状态管理、分类/分组展示、进度展示、登录与数据同步、模态框与确认弹窗
