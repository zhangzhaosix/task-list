# 每日任务清单

一个单页静态前端应用，用于记录、管理和同步每日任务。

## 项目介绍

这个项目把任务添加、状态管理、分类展示、进度查看和登录同步放在同一个页面里，适合快速记录当天要做的事情，减少切换成本。

## 功能

- 添加每日任务
- 标记任务完成状态
- 按分类或分组查看任务
- 查看进度统计
- Firebase 登录与 Firestore 数据同步
- 云端最近安全版本备份与恢复
- 模态框与确认弹窗交互
- 响应式布局，支持桌面和移动端

## 技术栈

- HTML
- CSS
- 原生 JavaScript
- Firebase Auth
- Firestore
- Google Fonts

## 目录结构

- `code/index.html`：页面入口，样式和脚本都在这个文件里
- `code/tests/`：回归测试
- `docs/PRODUCT.md`：产品与设计上下文
- `vercel.json`：Vercel 根路径路由配置
- `README.md`：项目说明与使用文档
- `LICENSE`：开源协议

## 本地启动方式

1. 直接用浏览器打开 `code/index.html`
2. 如果需要调试网络请求或跨域行为，可以用任意静态服务器临时打开

这个项目不需要安装依赖，也不需要构建步骤。

## 环境变量说明

当前版本不依赖额外环境变量。

Firebase 配置直接写在 `code/index.html` 的初始化代码中。如果需要切换 Firebase 项目，请修改页面里的 `firebase.initializeApp(...)` 配置。

## 部署注意事项

- Vercel 从仓库根目录部署，并通过 `vercel.json` 将根路径 `/` 映射到 `code/index.html`
- 其他静态托管平台可将 `code/` 设置为发布目录
- 如果使用 GitHub Pages，需要把 `code/` 的内容发布到 Pages 站点根目录，并保留 `.nojekyll`
- 部署前确认 `code/index.html` 中的 Firebase 配置已指向目标项目
- 不要上传密钥、私钥、日志、缓存和构建产物

## 常见问题

### 页面能打开，但数据不同步
先确认已登录 Firebase 账号，并检查网络、Firestore 权限规则和浏览器控制台报错。

### 页面空白或登录失败
先检查 Firebase 配置是否正确，浏览器是否能正常访问 Firebase CDN 和 Google Fonts。

### 修改后样式没有变化
确认浏览器没有缓存旧文件，必要时强制刷新。

### 如何恢复最近一次安全备份
登录后，如果账号存在云端备份，页脚会显示“恢复最近备份”按钮。确认后，页面会用最近一次安全版本覆盖当前任务、长期规划和思维模式。普通保存会通过 Firestore 事务保留最近一次有效版本；如果本次保存为空或数据格式异常，会继续保留上一个非空安全版本。
